# Support & issues

The **official issue tracker** for Retro Creator lives in this repository:

👉 **[github.com/Nelfe80/RetroCreator-Wiki/issues](https://github.com/Nelfe80/RetroCreator-Wiki/issues)**

## Reporting a bug

Open a [bug report](https://github.com/Nelfe80/RetroCreator-Wiki/issues/new?template=bug_report.yml)
and include:

- what you did, what you expected, what happened;
- your Windows version and streaming software (with version);
- whether APIExpose is running and its version;
- screenshots or a short clip when the problem is visual.

## Requesting a feature

Open a [feature request](https://github.com/Nelfe80/RetroCreator-Wiki/issues/new?template=feature_request.yml)
describing the problem you want solved — the streaming situation, not only the
solution you imagine. It helps us weigh it against the roadmap.

## Questions

Use [GitHub Discussions](https://github.com/Nelfe80/RetroCreator-Wiki/discussions) if
enabled, or open an issue with the `question` label.

!!! note
    Issues about APIExpose itself (the data layer) belong to the
    [APIExpose tracker](https://github.com/Nelfe80/RetroBat-APIExpose/issues); issues
    about the developer SDK belong to the
    [SDK tracker](https://github.com/Nelfe80/APIExpose-SDK/issues).
